# План тестирования парсера

## Типы тестирования

### 1. Модульное тестирование (Unit Testing)
- Тестирование отдельных методов Spider
- Тестирование вспомогательных функций
- Тестирование обработки данных

### 2. Интеграционное тестирование
- Тестирование взаимодействия компонентов
- Тестирование пайплайнов и middleware
- Тестирование сохранения данных

### 3. Функциональное тестирование
- Тестирование полного цикла парсинга
- Тестирование на реальных данных (ограниченное количество страниц)
- Проверка корректности выходных данных

### 4. Тестирование производительности
- Проверка соблюдения задержек между запросами
- Тестирование потребления памяти
- Тестирование времени выполнения

## Тестовые сценарии

### Сценарий 1: Базовый парсинг одной страницы
- Цель: Проверка корректности парсинга таблицы
- Вход: Страница 1 (https://smart-lab.ru/q/bonds/.../page1/)
- Ожидаемый результат: CSV файл с данными облигаций
- Проверки:
  - Файл создан
  - Заголовок присутствует
  - Количество строк соответствует таблице на странице
  - Данные корректны (нет пустых значений в ключевых полях)

### Сценарий 2: Пагинация (3 страницы)
- Цель: Проверка обработки нескольких страниц
- Вход: Страницы 1-3
- Ожидаемый результат: CSV файл с данными со всех страниц
- Проверки:
  - Все страницы обработаны
  - Задержки между запросами ≥5 секунд
  - Нет дубликатов данных
  - Прогресс логируется корректно

### Сценарий 3: Обработка ошибок
- Цель: Проверка устойчивости к ошибкам
- Входы:
  - Несуществующая страница (404)
  - Страница с измененной структурой
  - Медленный ответ (таймаут)
- Ожидаемый результат:
  - Ошибки логируются, но не останавливают парсер полностью
  - Парсер продолжает работу после обрабатываемых ошибок

### Сценарий 4: Сохранение данных
- Цель: Проверка корректности CSV файла
- Проверки:
  - Кодировка UTF-8
  - Разделитель запятая
  - Заголовок только один раз
  - Порядок столбцов соответствует исходной таблице
  - Отсутствуют последние 2 безымянных столбца

## Инструменты тестирования

### 1. pytest для модульных тестов
```python
import pytest
from bonds_parser.spiders.smartlab_bonds_spider import SmartLabBondsSpider

def test_extract_page_number():
    spider = SmartLabBondsSpider()
    assert spider.extract_page_number("/page1/") == 1
    assert spider.extract_page_number("/page42/") == 42
```

### 2. Scrapy Contracts для тестирования Spider
```python
class SmartLabBondsSpider(scrapy.Spider):
    # ... 
    
    def parse(self, response):
        """Парсинг страницы с облигациями
        @url https://smart-lab.ru/q/bonds/order_by_coupon_value/desc/page1/?paids_year=12
        @returns items 50 100  # Ожидается 50-100 облигаций на странице
        @returns requests 1  # Ожидается 1 запрос на следующую страницу
        """
```

### 3. Мокирование ответов для тестирования
```python
from scrapy.http import Response, Request, HtmlResponse

def test_parse_bonds_table():
    spider = SmartLabBondsSpider()
    
    # Создание мокового ответа с тестовым HTML
    html_content = """
    <table class="bonds">
        <tbody>
            <tr><td>1</td><td>Облигация 1</td>...</tr>
            <tr><td>2</td><td>Облигация 2</td>...</tr>
        </tbody>
    </table>
    """
    
    response = HtmlResponse(url="http://test.com", body=html_content.encode())
    bonds = spider.parse_bonds_table(response)
    
    assert len(bonds) == 2
    assert bonds[0]['number'] == '1'
```

## План выполнения тестирования

### Этап 1: Подготовка (1 час)
1. Настройка тестового окружения
2. Создание тестовых данных (моковые HTML страницы)
3. Настройка pytest конфигурации

### Этап 2: Модульное тестирование (2 часа)
1. Тестирование методов извлечения данных
2. Тестирование обработки пагинации
3. Тестирование вспомогательных функций
4. Тестирование обработки ошибок

### Этап 3: Интеграционное тестирование (2 часа)
1. Тестирование полного цикла Spider на моковых данных
2. Тестирование взаимодействия с Pipeline
3. Тестирование middleware (задержки, User-Agent)

### Этап 4: Функциональное тестирование (3 часа)
1. Тестирование на реальной странице (ограниченное количество)
2. Проверка корректности выходного CSV
3. Проверка соблюдения задержек
4. Тестирование обработки ошибок на реальных сценариях

### Этап 5: Регрессионное тестирование (1 час)
1. Повторное тестирование после изменений
2. Проверка критических функций

## Критерии успешного тестирования

### Обязательные критерии
1. Все модульные тесты проходят
2. Spider корректно парсит тестовые данные
3. CSV файл создается с правильной структурой
4. Задержки между запросами соблюдаются (≥5 секунд)
5. Ошибки обрабатываются и логируются

### Дополнительные критерии
1. Производительность: парсинг 100+ облигаций без ошибок
2. Устойчивость: работа при временных проблемах сети
3. Качество кода: соответствие PEP8, отсутствие утечек памяти

### Структура тестовой директории
```
tests/
├── unit/
│   ├── test_spider.py
│   ├── test_parsing.py
│   └── test_helpers.py

│   └── mock_responses.py

```

### Запуск тестов
```bash
# Все тесты
pytest tests/

# Только модульные тесты
pytest tests/unit/

# С отчетом о покрытии
pytest --cov=bonds_parser tests/

# Специфичный тест
pytest tests/functional/test_full_cycle.py -v
```

## Заключение
Тестирование парсера должно охватывать все ключевые аспекты:
- Корректность парсинга данных
- Обработку пагинации
- Соблюдение задержек
- Обработку ошибок
- Качество выходных данных

Рекомендуется начать с модульных тестов, затем перейти к интеграционным и завершить функциональным тестированием на ограниченном наборе реальных данных.
