### 3. Селекторы для парсинга

#### Таблица облигаций:
- Основная таблица: `table.simple-little-table.bonds` или `table.bonds`
- Заголовки: `thead tr th`
- Строки данных: `tbody tr`

#### Пагинация:
- Контейнер пагинации: `.pagination`
- Текущая страница: `.pagination .current`
- Ссылки на страницы: `.pagination a`
- Следующая страница: `.pagination a.next`

### 4. Столбцы для извлечения
На основе требований, нужно извлечь все столбцы кроме последних двух безымянных:

1. № (номер)
2. Имя (название облигации, может содержать ссылку)
3. Лет до погаш. (лет до погашения)
4. Доходн (доходность)
5. Купон (купонная ставка)
6. Цена (текущая цена)
7. Изменение (изменение цены)
8. Объем (объем торгов)
9. Дата погаш. (дата погашения)
10. Оферта (дата оферты)
11. НКД (накопленный купонный доход)
12. Дюрация (дюрация)
13. Выплаты в год (частота выплат)
14. Тип (тип облигации)
15. Эмитент (эмитент)

### 5. Особенности данных

#### Типы данных:
- Числовые значения: могут содержать проценты (%), десятичные разделители (запятая/точка)
- Даты: в формате DD.MM.YYYY
- Изменения: могут быть с плюсом/минусом, в процентах или абсолютных значениях
- Пустые значения: некоторые ячейки могут быть пустыми

#### Обработка:
- Очистка от лишних пробелов и символов
- Преобразование числовых значений (замена запятых на точки)
- Извлечение текста из ссылок (для имени облигации)
- Проверка на отсутствующие данные

### 6. Стратегия парсинга

#### Для каждой строки (`tr` в `tbody`):
1. Найти все ячейки (`td`)
2. Взять первые 15 ячеек (игнорировать последние 2)
3. Извлечь текст, очистить
4. Для ячейки с именем: извлечь текст ссылки если есть
5. Сохранить в структуру BondItem

#### Для пагинации:
1. Найти элемент пагинации
2. Проверить наличие ссылки на следующую страницу
3. Если есть - сформировать абсолютный URL
4. Если нет - завершить парсинг

### 7. Потенциальные проблемы

#### Структурные:
- Изменение классов таблицы
- Динамическая загрузка данных (AJAX)
- Разное количество столбцов на разных страницах
- Капча или блокировка при частых запросах

#### Данные:
- Разные форматы чисел (1,234.56 vs 1 234,56)
- Специальные символы в названиях
- Пустые или некорректные даты
- Изменение порядка столбцов

### 8. План валидации

1. Ручная проверка первой страницы
2. Сравнение количества извлеченных строк с видимым на странице
3. Проверка корректности данных (даты, числа)
4. Тестирование пагинации (минимум 3 страницы)
5. Проверка конечного CSV файла

### 9. Резервные селекторы

Если основной селектор не работает:
- Альтернативный 1: `table[class*="bonds"]`
- Альтернативный 2: `//table[contains(@class, 'bonds')]` (XPath)
- Альтернативный 3: Поиск таблицы по заголовкам (`th`, содержащим "Имя", "Доходн")
