## Содержание README.md

```markdown
# Парсер корпоративных облигаций Smart-Lab

Асинхронный парсер на Scrapy для сбора информации о корпоративных облигациях с сайта smart-lab.ru.

## Особенности

- Парсинг всех страниц с облигациями
- Сохранение в CSV с корректной структурой
- Задержки между запросами ≥5 секунд
- Имитация поведения реального пользователя
- Обработка ошибок и логирование
- Асинхронная реализация без блокировки GIL

## Требования

- Python 3.8+
- Scrapy 2.11+
- Дополнительные зависимости в requirements.txt

## Установка

1. Клонирование репозитория:
```bash
git clone <repository-url>
cd bonds_parser_AI
```

2. Создание виртуального окружения:
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. Установка зависимостей:
```bash
pip install -r requirements.txt
```

## Использование

### Базовый запуск
```bash
cd bonds_parser
scrapy crawl smartlab_bonds
```

### С ограничением страниц
```bash
scrapy crawl smartlab_bonds -a max_pages=5
```

### С ограничением облигаций
```bash
scrapy crawl smartlab_bonds -a max_bonds=100
```

### Сохранение в другой файл
```bash
scrapy crawl smartlab_bonds -o output/bonds.csv
```

## Конфигурация

### Настройки в settings.py
- `DOWNLOAD_DELAY`: Задержка между запросами (по умолчанию 5 секунд)
- `LOG_LEVEL`: Уровень логирования
- `CONCURRENT_REQUESTS`: Количество параллельных запросов

### Переменные окружения
- `SMARTLAB_PROXY`: Прокси для запросов
- `LOG_FILE`: Путь к файлу логов

## Выходные данные

Парсер создает CSV файл `data/bonds_output.csv` со следующей структурой:

| Столбец | Описание |
|---------|----------|
| № | Порядковый номер |
| Имя | Название облигации |
| Лет до погаш. | Лет до погашения |
| Доходн | Доходность |
| Купон | Купонная ставка |
| Цена | Текущая цена |
| Изменение | Изменение цены |
| Объем | Объем торгов |
| Дата погаш. | Дата погашения |
| Оферта | Дата оферты |
| НКД | Накопленный купонный доход |
| Дюрация | Дюрация |
| Выплаты в год | Частота выплат |
| Тип | Тип облигации |
| Эмитент | Эмитент |

## Логирование

Логи сохраняются в `logs/bonds_parser.log` и выводятся в консоль.

Уровни логирования:
- INFO: Прогресс работы, статистика
- WARNING: Ошибки соединения, проблемы с данными
- ERROR: Критические ошибки

## Обработка ошибок

Парсер обрабатывает:
- Таймауты соединения
- HTTP ошибки (404, 429, 500+)
- Отсутствие ожидаемых элементов на странице
- Ошибки записи данных

## Инструкции по развертыванию

### Локальное развертывание
```bash

pip install -r requirements.txt

# 2. Запуск парсера
scrapy crawl smartlab_bonds

# 3. Проверка результатов

### Docker развертывание
```dockerfile

WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["scrapy", "crawl", "smartlab_bonds"]
```

### Планировщик задач (cron)
```bash

```

## Руководство по устранению неполадок

### Проблема 1: Парсер не запускается
**Решение**: Проверьте установку Scrapy и зависимости

### Проблема 2: Нет данных в CSV файле
**Решение**: Проверьте логи на наличие ошибок парсинга

### Проблема 3: Блокировка IP
**Решение**: Увеличьте задержки, используйте прокси

### Проблема 4: Изменение структуры сайта
**Решение**: Обновите селекторы в spider
