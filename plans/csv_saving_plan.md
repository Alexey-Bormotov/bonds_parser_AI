# Краткий план сохранения данных в CSV

## Основные требования
1. Сохранение всех данных облигаций в CSV файл
2. Заголовок таблицы только один раз в начале файла
3. Соответствие порядка столбцов исходной таблице (исключая последние 2 безымянных столбца)
4. Кодировка UTF-8
5. Промежуточное сохранение не требуется (по желанию)

## Компоненты реализации

### 1. Item Pipeline (BondsCsvPipeline)
- Открытие файла при старте spider
- Запись заголовка один раз
- Постепенная запись данных по мере парсинга
- Закрытие файла при завершении

### 2. Настройки Scrapy
- Активация pipeline в settings.py
- Настройка экспорта (кодировка, разделитель)

### 3. Структура CSV файла
```
№,Имя,Лет до погаш.,Доходн,Купон,Цена,Изменение,Объем,Дата погаш.,Оферта,НКД,Дюрация,Выплаты в год,Тип,Эмитент
1,РЖД 001Р-02Р,2.5,12.34%,10.50%,98.75,+0.25%,1,234.56,15.09.2026,15.09.2024,15.23,2.1,2,Корп.,РЖД
...
```

```python
class BondsCsvPipeline:

    def process_item(self, item, spider):
        # Запись строки данных
    
    def close_spider(self, spider):
        # Закрытие файла
```

### Шаг 3: Настройка Scrapy
```python
ITEM_PIPELINES = {
    'bonds_parser.pipelines.BondsCsvPipeline': 300,
}
FEED_EXPORT_ENCODING = 'utf-8'
```

### Шаг 4: Интеграция со Spider
- Spider возвращает BondItem объекты
- Pipeline автоматически их обрабатывает
