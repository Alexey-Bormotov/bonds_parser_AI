# Парсер корпоративных облигаций Smart-Lab

Асинхронный парсер на Scrapy для сбора информации о корпоративных облигациях с сайта smart-lab.ru.

## Особенности

- Парсинг всех страниц с облигациями
- Сохранение в CSV с корректной структурой
- Задержки между запросами ≥5 секунд
- Имитация поведения реального пользователя
- Обработка ошибок и логирование
- Асинхронная реализация без блокировки GIL
- Принципы объектно-ориентированного программирования

## Требования

- Python 3.8+
- Scrapy 2.11+
- openpyxl 3.1+ (для конвертации CSV в XLSX)
- Дополнительные зависимости в requirements.txt

## Установка

1. Клонирование репозитория:
```bash
git clone <repository-url>
cd bonds_parser
```

2. Создание виртуального окружения:
```bash
python -m venv venv

# или
venv\Scripts\activate  # Windows
```

3. Установка зависимостей:
```bash
pip install -r requirements.txt
```

## Использование

### Базовый запуск
```bash
cd bonds_parser
scrapy crawl smartlab_bonds
```

### С ограничением страниц
```bash
scrapy crawl smartlab_bonds -a max_pages=5
```

### С ограничением облигаций
```bash
scrapy crawl smartlab_bonds -a max_bonds=100
```

### Сохранение в другой файл
```bash
scrapy crawl smartlab_bonds -o data/custom_output.csv
```

### Запуск с подробным логированием
```bash
scrapy crawl smartlab_bonds --loglevel=DEBUG
```

## Конфигурация

### Настройки в settings.py
- `DOWNLOAD_DELAY`: Задержка между запросами (по умолчанию 5 секунд)
- `RANDOMIZE_DOWNLOAD_DELAY`: Случайная добавка к задержке
- `CONCURRENT_REQUESTS`: Количество параллельных запросов (1 для соблюдения задержек)
- `LOG_LEVEL`: Уровень логирования (INFO, DEBUG, WARNING, ERROR)
- `ROBOTSTXT_OBEY`: Соблюдение robots.txt (False)

### Параметры командной строки
- `max_pages`: Максимальное количество страниц для парсинга (0 = все)
- `max_bonds`: Максимальное количество облигаций для сбора (0 = все)

## Выходные данные

Парсер создает CSV файл `data/bonds_output.csv` со следующей структурой (20 колонок, разделитель — точка с запятой):

| Столбец | Описание | Пример |
|---------|----------|--------|
| № | Порядковый номер | 1 |
| Имя | Название облигации | РЖД 001Р-02Р |
| *(пустая колонка 3)* | График (картинка) | |
| Лет до погаш. | Лет до погашения | 2.5 |
| Доходн | Доходность | 12.34% |
| Год.куп.дох. | Годовой купонный доход | 10.50% |
| Куп.дох.посл. | Купонный доход последний | 10.60% |
| Рейтинг | Кредитный рейтинг | A- |
| Объем, млн руб | Объем торгов в млн руб | 1,234.56 |
| Купон, руб | Купон в рублях | 15.23 |
| Частота, раз в год | Частота выплат купона | 2 |
| НКД, руб | Накопленный купонный доход в рублях | 15.23 |
| Дюр-я, лет | Дюрация в годах | 2.1 |
| Цена | Текущая цена | 98.75 |
| Дата купона | Дата ближайшего купона | 15.09.2026 |
| Размещение | Дата размещения | 15.09.2024 |
| Погашение | Дата погашения | 15.09.2026 |
| Оферта | Дата оферты | 15.09.2024 |
| *(пустая колонка 19)* | Кнопка добавления в портфель | |
| *(пустая колонка 20)* | Кнопка добавления в список наблюдения | |
**Примечание**: Колонки 3, 19 и 20 содержат графические элементы (картинки) и не парсятся, в CSV остаются пустыми. Разделитель колонок — точка с запятой (`;`), что соответствует формату экспорта сайта smart-lab.ru.

### Автоматическая конвертация в XLSX

После завершения работы spider автоматически создается XLSX файл (`data/bonds_output.xlsx`) с теми же данными, что и в CSV. Конвертация выполняется с помощью утилиты `bonds_parser/utils/csv_to_xlsx.py`, которая использует библиотеку `openpyxl`. Файл XLSX удобен для открытия в Microsoft Excel и других табличных редакторах.

**Новая функциональность**: В XLSX файле все облигации с рейтингом "AAA" автоматически подсвечиваются бледно-зелёным цветом (заливка `#C6EFCE`). Это позволяет визуально выделить наиболее надёжные облигации.

Для ручной конвертации можно запустить утилиту отдельно:

```bash
python -m bonds_parser.utils.csv_to_xlsx data/bonds_output.csv
```


## Архитектура

### Ключевые компоненты

1. **SmartLabBondsSpider** - основной spider для парсинга
   - Асинхронные методы парсинга
   - Обработка пагинации
   - Валидация данных
   - Обработка ошибок

2. **BondItem** - модель данных облигации
   - 20 полей, соответствующих всем колонкам таблицы на сайте (включая пустые)
   - Методы для преобразования в словарь и CSV
   - Порядок полей соответствует порядку колонок на сайте

3. **BondsCsvPipeline** - pipeline для экспорта
   - Сохранение в CSV с одним заголовком
   - Кодировка UTF-8
   - Постепенная запись данных

4. **Middleware** - для имитации пользователя
   - RandomUserAgentMiddleware - ротация User-Agent
   - DelayMiddleware - точные задержки между запросами
   - ErrorHandlingMiddleware - обработка HTTP ошибок

### Принципы реализации

1. **Асинхронность**: Все операции неблокирующие, используются async/await
2. **ООП**: Четкая структура классов, наследование, инкапсуляция
3. **DRY**: Повторное использование кода через базовые классы
4. **KISS**: Простота реализации и понимания
5. **PEP8**: Соблюдение стандартов кодирования Python

## Логирование

Логи сохраняются в `logs/bonds_parser.log` и выводятся в консоль.

Уровни логирования:
- **INFO**: Прогресс работы, статистика
- **WARNING**: Ошибки соединения, проблемы с данными
- **ERROR**: Критические ошибки
- **DEBUG**: Детальная информация для отладки

## Обработка ошибок

Парсер обрабатывает следующие типы ошибок:

1. **Ошибки соединения**: Таймауты, отказ DNS
2. **HTTP ошибки**: 404 (страница не найдена), 429 (Too Many Requests), 500+ (серверные ошибки)
3. **Ошибки парсинга**: Отсутствие ожидаемых элементов, некорректная структура HTML
4. **Ошибки записи**: Проблемы с файловой системой, кодировкой

## Производительность

- Задержка между запросами: 5-7 секунд (случайная добавка 0-2 секунды)
- Параллельные запросы: 1 (для соблюдения задержек)
- Потребление памяти: Оптимизировано для обработки больших объемов данных
- Время выполнения: Зависит от количества страниц (≈5 секунд на страницу)

## Ограничения

1. **Структура сайта**: Парсер зависит от текущей структуры HTML сайта smart-lab.ru
2. **Блокировка IP**: При слишком частых запросах возможна блокировка IP
3. **Динамический контент**: Не обрабатывает JavaScript-генерируемый контент

## Мониторинг

Во время работы парсер выводит информацию о прогрессе:
- Номер обрабатываемой страницы
- Количество найденных облигаций на странице
- Общее количество обработанных облигаций
- Время выполнения
